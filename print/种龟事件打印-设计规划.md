## 二、QR 码内容与权限方案（核心设计）

### 2.1 方案选型

| 方案 | 描述 | 可行性 |
|------|------|--------|
| A: 打印 wxacode 图片 | 云函数生成小程序码图片 → 下载 → 打印机打印图片 | ❌ lpapi SDK 不支持打印位图 |
| B: QR 码 + 微信 URL Link | 云函数生成 URL Link → lpapi `drawQrCode()` 打印 | ✅ **推荐** |
| C: QR 码 + 自定义 scheme | 编码 `petId:openid` 在 QR 码中 | ❌ 微信扫码不自动跳小程序 |

**结论：采用方案 B**。使用微信云开发 `cloud.openapi.urlLink.generate()` 生成永久 URL Link（如 `https://wxaurl.cn/xxxxxxxx`），然后通过 `lpapi.drawQrCode()` 将其打印在标签上。

### 2.2 URL Link 生成流程

```
┌──────────────────────────────────────────────────┐
│                printEventRecord()                 │
│  detail.js 中触发打印                              │
└─────────┬────────────────────────────────────────┘
          │ 1. 调用云函数，传入 petId + recordId
          ▼
┌──────────────────────────────────────────────────┐
│         cloudfunctions/qrcode/index.js            │
│  新增 action: 'generateUrlLink'                   │
│  cloud.openapi.urlLink.generate({                 │
│    path: 'pages/pet/detail',                     │
│    query: 'petId=xxx&recordId=yyy&from=scan'     │
│  })                                               │
│  → 返回 url_link (如 https://wxaurl.cn/xxxx)      │
└─────────┬────────────────────────────────────────┘
          │ 2. 返回 URL Link 字符串
          ▼
┌──────────────────────────────────────────────────┐
│                lpapi 打印                          │
│  drawQrCode({ text: urlLink, x:2, y:2, w:16 })   │
│  drawText(...) 绘制右侧文字                        │
│  commitJob()                                      │
└──────────────────────────────────────────────────┘
```

### 2.3 权限校验流程（扫码后）

```
用户扫描标签上的 QR 码
    │
    ▼
微信打开 URL Link → 跳转小程序 pages/pet/detail?petId=xxx&recordId=yyy&from=scan
    │
    ▼
detail.js onLoad(options)
    │
    ├─ options.from === 'scan' ?
    │   │
    │   ├─ YES → 进入扫码模式
    │   │   │
    │   │   ├─ 调云函数获取 pet 数据
    │   │   ├─ 比对 currentUser.openid === pet._openid ?
    │   │   │   ├─ YES → 正常编辑模式，自动定位到 recordId 对应记录
    │   │   │   └─ NO  → 显示 "仅创建者可编辑" 提示，进入只读模式
    │   │   │
    │   └─ NO → 正常从列表进入，保持现有逻辑不变
```

**关键点**：
- `pet._openid`（微信云数据库自动字段）即创建人的 openid
- 比对逻辑放在 `detail.js` 的 `onLoad` / `loadPetDetail` 中
- 权限不足时：隐藏编辑按钮、底部操作栏，仅展示「种龟事件」+「家族谱系」只读视图

### 2.4 云函数改动清单

**qrcode/index.js — 新增 action**：

```javascript
// 输入: { petId, recordId }
// 调用: cloud.openapi.urlLink.generate({ path, query })

```

## 三、标签打印布局设计

### 3.1 视觉布局（40×20mm）

```
┌──────────────────────┬────────────────────────────┐
│                      │  交配                      │
│                      │  2026-06-08 与三号线公龟交配  │
│    ██ QR CODE ██     │                            │
│    16×16mm          │  2026-06-08 17:00           │
│                      │  龟上心                     │
│                      │                            │
└──────────────────────┴────────────────────────────┘
  ← 18mm →              ← 22mm →
```

### 3.2 实际布局（已实施）

| 元素 | API | 参数 |
|------|-----|------|
| 开始任务 | `startJob` | `{ width: 40, height: 20, jobName, gapType: 2 }` |
| QR 码 | `drawImage` (base64) | `{ image: cleanBase64, x: 1, y: 1, width: 16, height: 16 }` |
| 记录内容 | `drawText` | `{ text: content(≤40字), fontHeight: 3, x: 19, y: 3, width: 19, height: 10 }` |
| 记录时间 | `drawText` | `{ text: 'yyyy-MM-dd HH:mm', fontHeight: 2, x: 19, y: 13, width: 19 }` |

> SDK 不支持 `drawQrCode`，采用 `drawImage` 直接接收 base64 编码的 QR 码 PNG 图片。云函数中使用 `qrcode` npm 包生成 QR 码。

## 四、核心技术要点（已实施）

### 4.1 QR 码生成流程
1. 云函数 `qrcode` 的 `generateUrlLink` action 调用 `cloud.openapi.urlLink.generate` 生成永久 URL Link
2. 同一 action 中用 `qrcode` npm 包生成 QR 码并返回 base64 PNG
3. 小程序端接收后直接传给 `lpapi.drawImage({ image: base64Data, ... })`

### 4.2 打印入口
- **记录详情弹窗**：点击种龟事件卡片 → 弹窗 → 点击「打印」→ `printRecord()`
- **添加记录弹窗**：填写内容 → 点击「打印」→ `printNewRecord()` → 自动保存 + 生成QR + 打印

### 4.3 权限校验（扫码后）
- `detail.wxml` 底部操作栏：`wx:if="{{!isPublicMode && !isReadOnly}}"`
- 提醒完成按钮：`wx:if="{{!isReadOnly}}"` 隐藏
- `detail.js onLoad`：检查 `options.from === 'scan'` → 设置 `isReadOnly: true`
- `loadPetDetail`：比对 `openid === pet._openid` → 匹配则 `isReadOnly: false`
- `confirmAddRecord` 和 `printNewRecord` 都加了 `_isSavingRecord` 互斥锁防止重复保存
