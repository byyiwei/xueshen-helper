# 推广返利（Referral / Rebate）功能模块实施计划

## 一、需求摘要

为用户提供推广获客能力：通过专属邀请链接邀请新用户注册并完成首单付费消费后，邀请人获得现金佣金。后台支持佣金比例配置、提现审核与数据统计，形成裂变增长闭环。

### 已确认决策

| 决策项 | 结论 |
|---|---|
| 佣金形式 | 现金（元），提现到支付宝/微信，**非**题数点数 |
| 首单判定 | 仅 `price > 0` 的付费订单计入首单（免费套餐不计） |
| 邀请链接 | 每用户生成 8 位随机邀请码，链接 `/invite?code=xxxx` |
| 脚本菜单 | 油猴脚本首页加"推广"按钮，`window.open` 跳转网页推广页（不内嵌 Tab） |

### 补充技术决策

- **佣金结算模型**：pending（冷却期）→ paid 双状态。订单完成先写 `pending`，冷却期（`referral_settle_days`，默认 7 天）过后惰性结算转 `paid` 并入余额，防退款套利。
- **提现托管**：提交提现原子扣减余额；驳回退还余额；通过后管理员线下打款。
- **收款信息**：新增 `user_payment_info` 表预存支付宝/微信账号+二维码，提现表单一键带入。
- **二维码存储**：base64 data-URI 存 TEXT 列，复用 avatar 校验范式（`data:image/` 前缀，≤1MB）。

## 三、数据库改动（src/database.py）

### 3.1 新建 4 张表（加入 `_init_tables` 末尾，约 line 350 之后）

```sql
-- 邀请关系表
CREATE TABLE IF NOT EXISTS referrals (
    id INT PRIMARY KEY AUTO_INCREMENT,
    inviter_username VARCHAR(50) NOT NULL COMMENT '邀请人',
    invitee_username VARCHAR(50) NOT NULL COMMENT '被邀请人',
    invite_code VARCHAR(16) NOT NULL COMMENT '使用的邀请码',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_invitee (invitee_username),
    INDEX idx_inviter (inviter_username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4

-- 佣金记录表
CREATE TABLE IF NOT EXISTS commission_logs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    inviter VARCHAR(50) NOT NULL,
    invitee VARCHAR(50) NOT NULL,
    order_no VARCHAR(64) NOT NULL,
    order_amount DECIMAL(10,2) DEFAULT 0,
    rate DECIMAL(5,4) DEFAULT 0 COMMENT '费率 0.0000-1.0000',
    commission_amount DECIMAL(10,2) DEFAULT 0,
    status VARCHAR(20) DEFAULT 'pending' COMMENT 'pending/paid',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_order (order_no),
    INDEX idx_inviter_status (inviter, status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4

-- 提现申请表
CREATE TABLE IF NOT EXISTS withdrawals (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    pay_method VARCHAR(20) NOT NULL COMMENT 'alipay/wechat',
    pay_account VARCHAR(255) NOT NULL,
    qr_code_path TEXT COMMENT '收款二维码 data-URI',
    status VARCHAR(20) DEFAULT 'pending' COMMENT 'pending/approved/rejected',
    reject_reason VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    reviewed_at TIMESTAMP NULL DEFAULT NULL,
    INDEX idx_username_status (username, status),
    INDEX idx_status_created (status, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4

-- 用户收款信息表（预存，提现时带入）
CREATE TABLE IF NOT EXISTS user_payment_info (
    username VARCHAR(50) PRIMARY KEY,
    alipay_account VARCHAR(255) DEFAULT '',
    alipay_qr TEXT,
    wechat_account VARCHAR(255) DEFAULT '',
    wechat_qr TEXT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

### 3.4 新增 DB 方法

- `_gen_unique_invite_code()` — 字符集 `[A-Z2-9]`（去 0/O/1/I），8 位，20 次重试 + UNIQUE 兜底
- `get_or_create_invite_code(username)` — 老用户懒生成
- `_bind_referral(invitee, invite_code)` — 防自邀 + 防重复 + 链路回溯 10 层防环
- `_settle_first_order_commission(invitee, order_no, amount)` — 查邀请关系 + 防重复 + 写 pending 佣金
- `settle_pending_commissions(inviter)` — 惰性结算：超冷却期的 pending 转 paid 并加余额（并发安全用 `WHERE status='pending'` + rowcount 校验）
- `get_referral_profile(username)` — 返回邀请码/邀请人数/已结算/待结算/余额
- `get_user_payment_info` / `save_user_payment_info` — 收款信息 CRUD
- `create_withdrawal(username, amount, pay_method, pay_account, qr)` — 原子扣余额 + 防重复 pending
- `list_user_withdrawals` / `list_admin_withdrawals` — 列表查询
- `approve_withdrawal(wid)` / `reject_withdrawal(wid, reason)` — 审核（reject 退还余额）
- `referral_stats()` — Top10 邀请人 + 总佣金统计

## 八、反滥用与边界条件

| 风险 | 防御措施 | 位置 |
|---|---|---|
| 自邀 | `inviter == invitee` 拒绝 | `_bind_referral` |
| 重复绑定 | `referrals.uk_invitee` 唯一索引 + 查重 | DDL + `_bind_referral` |
| 环形邀请 | 链路回溯 10 层 | `_bind_referral` |
| 重复佣金 | `commission_logs.uk_order` 唯一索引 | DDL + `_settle_first_order_commission` |
| 余额超额提现 | 原子 `WHERE commission_balance >= amount` + rowcount | `create_withdrawal` |
| 重复提现 | 查 pending 记录拒绝 | `create_withdrawal` |
| 提现金额下限 | `>= min_withdraw` 校验 | do_POST withdraw |
| 二维码过大 | `len(qr) <= 1MB` + `data:image/` 前缀 | do_POST |
| 退款套利 | 冷却期 pending 不入余额 | `settle_pending_commissions` |
| 驳回后余额 | reject 时退还 | `reject_withdrawal` |
| 并发扣余额竞态 | 原子 UPDATE + rowcount 检查 | `create_withdrawal` |
| 免费订单触发 | `if price > 0` 才进佣金逻辑 | `apply_paid_order` |
| 邀请码碰撞 | 20 次重试 + UNIQUE 索引 | `_gen_unique_invite_code` |
