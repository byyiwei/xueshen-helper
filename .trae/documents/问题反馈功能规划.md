# 问题反馈功能实施计划

## 概述

在 user.html 新增"问题反馈"菜单，用户可提交问题反馈；admin.html 新增对应菜单查看所有反馈，管理员处理后通过邮件回复用户。

## 现状分析

### 技术栈
- **user.html**：原生 JS（非 Vue），菜单通过 `data-section` 切换，`switchSection(name)` 函数控制
- **admin.html**：Vue 3，二级折叠菜单分组（推广返利/支付财务/日志监控/系统配置）
- **后端**：Python `BaseHTTPRequestHandler`，`do_GET`/`do_POST` 路由，`_get_user_from_token()` 获取用户
- **邮件**：`send_email(to_addr, subject, body_text, body_html, template_id, scene, variables)` 已实现（backend.py:839）
- **数据库**：MySQL，`CREATE TABLE IF NOT EXISTS` 模式，`_ensure_index` 辅助方法

### 关键文件
- `src/database.py`：数据库表定义和操作方法
- `src/backend.py`：HTTP 路由和业务逻辑
- `static/user.html`：用户前台页面
- `static/admin.html`：管理后台页面

## 实施方案

### 1. 数据库新增 `user_feedback` 表

**文件**：`src/database.py`

在 `email_templates` 表后面添加：

```sql
CREATE TABLE IF NOT EXISTS user_feedback (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL COMMENT '提交用户',
    email VARCHAR(100) DEFAULT '' COMMENT '联系邮箱(默认取用户注册邮箱)',
    category VARCHAR(30) NOT NULL DEFAULT 'other' COMMENT '反馈类型: bug/feature/payment/account/other',
    title VARCHAR(200) NOT NULL COMMENT '问题标题',
    content TEXT NOT NULL COMMENT '问题描述',
    attachment TEXT DEFAULT '' COMMENT '附件URL(JSON数组,暂不支持上传,预留)',
    status VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT '状态: pending/processing/resolved/closed',
    admin_reply TEXT COMMENT '管理员回复内容',
    replied_at TIMESTAMP NULL COMMENT '回复时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

新增数据库方法：
- `create_feedback(username, email, category, title, content)` — 用户提交反馈
- `list_user_feedback(username, limit=20)` — 用户查看自己的反馈
- `list_feedback_admin(status, category, keyword, page, page_size)` — 管理员查看所有反馈（支持筛选/分页）
- `get_feedback_by_id(id)` — 获取单条反馈详情
- `update_feedback_status(id, status)` — 更新状态
- `reply_feedback(id, reply_text)` — 保存管理员回复

### 2. 后端新增 API 路由

**文件**：`src/backend.py`

#### 用户端 API

**POST `/api/user/feedback`** — 提交反馈
- 认证：`_get_user_from_token()`
- 参数：`category`, `title`, `content`（email 自动取用户邮箱，前端也可传入）
- 返回：`{code:200, msg:"提交成功"}`

**GET `/api/user/feedback`** — 查看我的反馈
- 认证：`_get_user_from_token()`
- 返回：`{code:200, feedback:[...]}`

#### 管理员端 API

**GET `/admin/feedback`** — 反馈列表
- 认证：`_check_admin()`
- 参数：`status`, `category`, `keyword`, `page`, `page_size`
- 返回：`{code:200, data:{rows, total, page, total_pages}}`

**GET `/admin/feedback/<id>`** — 反馈详情
- 返回：`{code:200, feedback:{...}}`

**POST `/admin/feedback/reply`** — 回复反馈（发邮件）
- 参数：`id`, `reply_text`
- 逻辑：
  1. 获取反馈详情和用户邮箱
  2. 调用 `send_email()` 发送回复邮件
  3. 保存回复内容到 `admin_reply` 字段
  4. 更新状态为 `resolved`
- 返回：`{code:200, msg:"回复成功"}` 或错误信息

**POST `/admin/feedback/status`** — 更新状态
- 参数：`id`, `status`
- 返回：`{code:200, msg:"更新成功"}`

#### 页面区域
在 `section-promotion` 后添加 `<section class="user-section" id="section-feedback">`：

- 反馈类型下拉：Bug 反馈 / 功能建议 / 支付问题 / 账号问题 / 其他

#### JS 逻辑
- `switchSection` 中添加 `if (name === 'feedback') loadFeedback();`
- `loadFeedback()` — 加载用户已提交的反馈列表
- `submitFeedback()` — 提交反馈表单，校验后 POST 到后端

### 5. 邮件回复

复用现有 `send_email()` 函数：
```python
send_email(
    to_addr=user_email,
    subject=f"[学神助手] 您的问题反馈回复：{feedback['title']}",
    body_text=reply_text,
    body_html=f"<html><body><p>您好 {username}，</p><p>您提交的问题反馈「{title}」已处理：</p><blockquote>{reply_text}</blockquote><p>如有疑问请继续反馈。</p></body></html>"
```

不使用模板系统，直接传入 subject/body_text/body_html，因为回复内容是管理员手动输入的。

## 假设与决策

1. **不增加附件上传功能**：用户反馈暂不支持上传截图/附件，预留 `attachment` 字段供未来扩展
2. **反馈类型固定 5 种**：bug/feature/payment/account/other，覆盖主要场景
3. **邮件回复而非站内回复**：管理员回复后直接发邮件到用户注册邮箱，同时页面也展示回复内容
4. **反馈放在系统配置组**：admin 侧"问题反馈"菜单放在"系统配置"二级菜单组下，与题库中心/模型配置等同级
5. **用户可多次提交反馈**：不做频率限制，但单条内容有字数限制
6. **状态流转**：pending → processing → resolved → closed，管理员可手动切换任意状态
