## 🔴 严重风险（必须修复）

### 1. database.py — MySQL 生产数据库明文密码

**文件**：`database.py` 第 19-29 行
**问题**：
```python
MYSQL_CONFIG = {
    "host": "43.138.147.202",      # 生产服务器公网IP
    "user": "xuexitong",
    "password": "199975yiwei",      # 明文密码！
    "database": "xuexitong",
}
```
**风险**：任何人都能用这个密码直连你的 MySQL 数据库，读取/篡改/删除全部用户数据、密码、充值记录。
**修复**：改为环境变量读取，见下方修复方案。

### 2. jwt_secret.key — JWT 签名密钥

**文件**：`jwt_secret.key`
**问题**：文件内容为 JWT 签名密钥明文
```
CZLmd0JULum6W0Ao4Z9qfGBxkz8jSUSEbhTkLEuSZBfsMcUw0oYzd7bZTF-eel0w
```
**风险**：任何人拿到此密钥就能伪造任意用户的认证 Token，以任意用户身份登录，包括管理员。
**修复**：加入 .gitignore，不提交到仓库。

### 3. config.json — 管理员密码哈希

**文件**：`config.json` 第 2 行
**问题**：
```json
"password": "pbkdf2$100000$ee3cba4e...$5d63d9305f3a..."
```
**风险**：虽然不是明文，但暴露了管理员密码的 PBKDF2 哈希值，可被离线暴力破解。
**修复**：加入 .gitignore，提供模板文件。

## ⚠️ 如果已经提交过敏感信息

如果之前已经提交过包含密码的版本，即使后续删除，Git 历史中仍会保留：

```bash
# 1. 使用 BFG 清理历史
bfg --replace-text passwords.txt
git reflog expire --expire=now --all
git gc --prune=now --grouptop

# 2. 或者直接重新初始化仓库
rm -rf .git
git init
git add -A
git commit -m "Initial commit"
git remote add origin <your-repo>
git push -u origin main
```

**同时必须**：修改 MySQL 密码、重新生成 JWT 密钥、修改管理员密码，因为旧的已经泄露。
