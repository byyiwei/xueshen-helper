### 1. 核心设计体系：奶油极简与温暖档案
项目采用了一套名为 **Design System v3** 的自定义视觉规范，核心理念为“奶油极简 · 温暖档案”。该体系通过 `miniprogram/styles/design.wxss` 集中管理，旨在营造柔和、专业且富有亲和力的宠物档案管理体验。

- **色彩哲学**：
  - **主色调**：以暖金色/琥珀色 (`#E8A400`) 为核心，象征档案的珍贵与时间的沉淀。
  - **背景色**：使用米白/奶油色系 (`#FAF8F5`, `#FFFCF5`) 替代纯白，减少视觉疲劳，提升质感。
  - **功能色**：定义了明确的性别色（父系蓝 `#5B8DEF` / 母系粉 `#E87DA0`）和状态色（成功绿、警告黄、危险红），并配套了低饱和度的背景色用于标签和卡片。
  - **中性色**：深灰 (`#1A1A1A`) 用于正文，浅灰 (`#999999`, `#BFBFBF`) 用于辅助信息，确保层级清晰。

- **设计令牌 (Design Tokens)**：
  - **圆角系统**：提供从 `6rpx` (xs) 到 `9999rpx` (full/pill) 的五级圆角规范，卡片通常使用 `20rpx` (xl) 以体现圆润感。
  - **阴影系统**：采用极轻的多层阴影 (`shadow-sm/md/lg`)，强调卡片的悬浮感而非边框分割。
  - **字体系统**：基于 `rpx` 单位定义了从 `22rpx` (mini) 到 `38rpx` (title) 的字号阶梯，行高统一为 `1.6`。

### 2. CSS 方法论与架构
项目基于微信小程序原生框架，采用 **全局样式 + 局部覆盖 + 工具类** 的混合模式。

- **全局基础层 (`app.wxss`)**：
  - 引入 `iconfont` 图标字体库。
  - 定义全局 Flex 布局工具类 (`.flex-center`, `.flex-between`)。
  - 声明旧版 CSS 变量（如 `--primary-color: #3A7CFF`），部分页面仍在使用此蓝色系主题。
  - 提供通用按钮样式 (`.btn-primary`, `.btn-secondary`) 和交互动画 (`.fade-in`, `.tap-scale`)。

- **设计规范层 (`styles/design.wxss`)**：
  - 通过 `@import` 在业务页面中引入。
  - 封装了高频组件类：`.card` (卡片容器), `.btn` (按钮系列), `.tag` (标签系列), `.chip` (筛选胶囊), `.nav-bar` (导航栏)。
  - 包含骨架屏动画 (`.skeleton`) 和空状态样式 (`.empty-box`)。

- **业务定制层**：
  - 各页面 `.wxss` 文件在引入设计规范后，针对特定布局进行微调。例如 `pages/pet/index.wxss` 实现了复杂的网格布局、毛玻璃效果叠加层以及悬浮操作按钮 (FAB)。

### 3. 关键视觉组件与交互细节
- **自定义 TabBar**：
  - 位于 `custom-tab-bar/`，采用 **毛玻璃 (Glassmorphism)** 效果 (`backdrop-filter: blur(20px)` )。
  - 图标使用纯 CSS/SVG 路径绘制（如爪印、房子、人形），选中态通过颜色填充和位移动画增强反馈。
  - 底部适配安全区 (`safe-area-inset-bottom`)。

- **卡片与列表**：
  - 宠物卡片采用大图覆盖式设计，左下角叠加半透明毛玻璃信息栏显示品种/性别，右上角显示性别标签。
  - 谱系图采用树状结构可视化，通过 CSS 伪元素绘制连接线。

- **动态反馈**：
  - 广泛使用 `transform: scale(0.96)` 和 `opacity` 变化作为点击反馈。
  - 加载状态使用旋转 Spinner 动画。

### 4. 预览与多端一致性
- **设计预览 (`design-preview/`)**：
  - 提供了一套与小程序视觉高度一致的 HTML/CSS 原型，用于在浏览器中快速验证 UI 改版。
  - 模拟了 iPhone 外壳和状态栏，确保移动端视觉还原度。

- **HTML 转图像服务**：
  - `utils/theme.js` 中内置了庞大的 HTML/CSS 生成逻辑，用于将宠物档案渲染为图片。
  - 该逻辑硬编码了与小程序端一致的样式（如渐变背景、圆角、阴影），确保分享海报与 App 内视觉统一。

### 5. 开发者规范
1. **单位使用**：所有尺寸必须使用 `rpx` 以确保在不同宽度屏幕上的自适应。
2. **样式引入**：新页面应优先 `@import "/styles/design.wxss"` 并使用其提供的工具类，避免重复定义基础样式。
3. **色彩约束**：严禁使用硬编码的颜色值，应从 `design.wxss` 的 CSS 变量中选取（如 `var(--c-primary)`）。
4. **图标规范**：优先使用 CSS 绘制的矢量图标或 `iconfont`，减少图片资源加载。
5. **兼容注意**：目前存在两套色彩体系（`app.wxss` 的蓝色系 vs `design.wxss` 的金色系），新开发功能应统一向 `design.wxss` (v3) 靠拢。
