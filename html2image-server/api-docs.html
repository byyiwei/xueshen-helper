HTML-to-Image Server · API 文档

通过 HTTP 接口将任意 HTML 内容渲染为 PNG / JPEG / WebP 图片 —— 基于无头 Chromium（Puppeteer）。

服务地址：http://localhost:3000

快速开始

  • Windows：双击或运行 start-server.bat；停止运行 stop-server.bat
  • Linux / macOS：chmod +x *.sh && ./start-server.sh；停止 ./stop-server.sh
  • 前台调试：node server.js；停止：Ctrl+C 或 node stop.js
  • 调用 POST /，JSON 请求体：{ "html": "<h1>Hello</h1>", "options": { ... } }
  • 响应中 image 字段是 base64 图片内容
  • 健康检查：GET /health   配置查看：GET /config

接口列表

POST / 主接口 · HTML → 图片

接收 HTML 内容并渲染成图片，以 base64 字符串返回。

请求体字段

字段类型必填默认值说明
htmlstring是—完整 HTML 文档或片段，例如 <h1 style='color:red'>Hello</h1>
optionsobject否{}渲染选项（见下表）

options 渲染选项

选项类型默认值说明
widthnumber1280视口宽度（CSS 像素，范围 100–10000）
heightnumber800视口高度（CSS 像素，范围 100–10000）
deviceScaleFactornumber2像素比 / DPR。数值越大图片越清晰（1–5）
formatstringpngpng / jpeg / webp
qualitynumber901–100。仅对 jpeg / webp 生效
fullPagebooleanfalse为 true 时截图整个可滚动页面
clipobject—{ x, y, width, height }（CSS 像素）— 仅截取指定区域
waitFornumber0加载完成后再等待的毫秒数（上限 30000）
loadTimeoutnumber30000页面加载超时（毫秒，下限 10000）

请求示例（curl）

# 最基本的 PNG 截图
curl -X POST http://localhost:3000/ \
  -H "Content-Type: application/json; charset=utf-8" \
  -d '{
    "html": "<h1 style='color:#0ea5e9'>Hello html2image</h1>",
    "options": {
      "width": 800,
      "height": 600,
      "format": "png",
      "deviceScaleFactor": 2
    }
  }'

请求示例（Node.js）

const res = await fetch('http://localhost:3000/', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    html: '<h1 style="color:#0ea5e9">Hello</h1>',
    options: { width: 800, height: 600, format: 'png' }
  })
});
const data = await res.json();
if (data.success) {
  const buf = Buffer.from(data.image, 'base64');
  require('fs').writeFileSync('output.png', buf);
}

成功响应

HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8

{
  "success": true,
  "image": "iVBORw0KGgoAAAANSUhEUgAA...",  // base64 图片数据
  "format": "png",
  "time": 1187                              // 总耗时（毫秒）
}

错误响应

400 JSON 解析失败  ·  413 请求体超过最大限制（默认 10MB） ·  500 浏览器启动 / 渲染异常 

{
  "success": false,
  "error": "Browser launch timeout (45000ms)"
}
GET /health 健康检查

轻量健康检查，适用于负载均衡 / 存活探针 / 监控告警。

{
  "success": true,
  "status": "ok",
  "browser": "idle",   // "running" 或 "idle"
  "uptime": "42s",
  "config": { "port": 3000, "host": "0.0.0.0" }
}
GET /config 只读配置摘要

返回当前生效的配置（不含敏感字段），便于排查问题。

{
  "success": true,
  "server": { "host": "0.0.0.0", "port": 3000 },
  "rendering": {
    "defaultViewport": { "width": 1280, "height": 800, "deviceScaleFactor": 2 },
    "defaultFormat": "png", ...
  },
  "http": { "maxRequestBodyBytes": 10485760 },
  "browser": { "headless": true, "launchTimeoutMs": 45000, ... }
}
GET / 服务信息

返回服务基本信息与链接。

{
  "success": true,

  "docs": "/api-docs",

启动与停止

Windows

:: 启动
start-server.bat

:: 停止
stop-server.bat

:: 或直接用 node 前台启动 / 停止
node server.js
node stop.js

Linux / macOS

# 首次运行：给脚本加上可执行权限（只需一次）
chmod +x start-server.sh stop-server.sh

# 启动（后台运行）
./start-server.sh

# 停止
./stop-server.sh

配置（config.json）

所有可配置项集中在项目根目录 config.json。修改后重启服务即可生效。

路径默认值说明
server.host0.0.0.0监听地址（0.0.0.0 表示所有网卡均可访问）
server.port3000HTTP 端口
process.pidFile.app.pid运行进程 PID 写入的文件路径（脚本读取用）
logging.logDirlogs日志输出目录
logging.stdoutLogapp_output.log标准输出日志文件名
browser.executablePath""（空 = 自动探测）强制使用某个 Chrome / Chromium 可执行路径
browser.headlesstrue是否无头模式（true 推荐）
browser.launchTimeoutMs45000浏览器启动超时（毫秒）
browser.protocolTimeoutMs30000Chrome DevTools 协议超时（毫秒）
rendering.defaultViewport1280×800 @2x默认视口
rendering.defaultFormatpng未指定 format 时的默认图片格式
rendering.defaultQuality90jpeg / webp 默认质量
rendering.loadTimeoutMs30000页面加载超时（毫秒）
rendering.fullPageByDefaultfalse是否默认整页截图
http.maxRequestBodyBytes10485760 (10MB)请求体最大字节数
http.startupWaitSeconds15启动脚本等待 .app.pid 出现的最长秒数

环境变量覆盖：任意配置项均可通过 H2I_<SECTION>_<KEY> 形式的环境变量覆盖。嵌套层级用单下划线分隔，key 内的下划线用双下划线表示。例如：

# 临时改端口为 8080（Windows CMD）
set H2I_SERVER_PORT=8080
node server.js

# Linux / macOS 一行
H2I_SERVER_PORT=8080 ./start-server.sh

# 指定 Chrome 可执行路径
H2I_BROWSER__EXECUTABLE_PATH=/usr/bin/chromium node server.js

# 调整浏览器启动超时为 60 秒
H2I_BROWSER__LAUNCH_TIMEOUT_MS=60000 node server.js

注意事项

  • 请求体最大 10 MB。
  • 首次调用 POST / 会稍慢 —— Chromium 是懒启动的。
  • HTML 中引用的外部资源（图片、字体）必须能被服务器所在网络访问。
  • 日志写入 logs/ 目录，按日期滚动。
  • 如需指定 Chrome 可执行文件路径，设置 browser.executablePath 或环境变量 H2I_BROWSER__EXECUTABLE_PATH。
  • 端口可通过配置或环境变量修改：PORT=8080 ./start-server.sh。
