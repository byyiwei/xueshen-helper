# 服务器部署环境变量配置

## 修改后的启动方式

`database.py` 已改为从环境变量读取数据库配置，需要在启动后端时设置环境变量。

### 方式一：systemd 服务（推荐）

编辑 `/etc/systemd/system/xs-backend.service`，在 `[Service]` 段添加环境变量：

```ini
[Service]
Type=simple
WorkingDirectory=/www/wwwroot/xuexitong
ExecStart=/usr/bin/python3 run_backend.py
Environment=MYSQL_HOST=127.0.0.1
Environment=MYSQL_PORT=3306
Environment=MYSQL_USER=xuexitong
Environment=MYSQL_PASSWORD=你的实际密码
Environment=MYSQL_DATABASE=xuexitong
Environment=JWT_SECRET=你的JWT密钥
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

重载并重启：
```bash
systemctl daemon-reload
systemctl restart xs-backend
```

### 方式二：宝塔进程守护管理器

在进程守护管理器中，编辑 xs-backend 任务的启动命令为：

```bash
MYSQL_HOST=127.0.0.1 MYSQL_PASSWORD=你的实际密码 JWT_SECRET=你的JWT密钥 python3 /www/wwwroot/xuexitong/run_backend.py
```

### 方式三：.env 文件（需要 python-dotenv）

如果不想改 systemd 配置，可以在项目目录创建 `.env` 文件：

```bash
cat > /www/wwwroot/xuexitong/.env << 'EOF'
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=xuexitong
MYSQL_PASSWORD=你的实际密码
MYSQL_DATABASE=xuexitong
JWT_SECRET=你的JWT密钥
EOF
chmod 600 /www/wwwroot/xuexitong/.env
```

然后在 `run_backend.py` 开头添加：
```python
# 加载 .env 文件
from pathlib import Path
env_file = Path(__file__).parent / ".env"
if env_file.exists():
    for line in env_file.read_text().splitlines():
        line = line.strip()
        if line and "=" in line and not line.startswith("#"):
            k, v = line.split("=", 1)
            os.environ.setdefault(k.strip(), v.strip())
```

## 当前已有的 jwt_secret.key

服务器上已存在 `jwt_secret.key` 文件，即使不设 `JWT_SECRET` 环境变量，后端也会从文件读取。所以 JWT 密钥不需要强制设置环境变量。

但 MySQL 密码**必须**设置环境变量，否则数据库连接会失败（密码为空）。
