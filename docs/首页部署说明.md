# 首页部署说明

## 目标
将 `publish_materials/index.html` 部署为 `https://xs.openget.cn/` 的首页。

---

## 部署步骤（通过宝塔面板）

### 第一步：上传文件

在宝塔文件管理中，进入 `/www/wwwroot/xuexitong/` 目录，创建 `intro` 文件夹。

然后上传以下文件：

```
/www/wwwroot/xuexitong/
└── intro/
    ├── index.html      ← 上传 publish_materials/index.html
    └── images/          ← 上传 publish_materials/images/ 整个文件夹
        ├── hero_clean.jpg
        ├── hero_banner.jpg
        ├── feature_ai_answer.jpg
        ├── feature_autoplay.jpg
        └── feature_dashboard.jpg
```

### 第二步：更新 Nginx 配置

在宝塔面板中，找到 `xs.openget.cn` 站点的 Nginx 配置，用 `publish_materials/xs.openget.cn.conf` 的内容覆盖。

新增了两条规则（在 `location /libs/` 之前）：

```nginx
# 0. 首页 - 产品介绍页（静态文件，不走后端）
location = / {
    root /www/wwwroot/xuexitong/intro;
    try_files /index.html =404;
}

# 0.1 产品介绍页图片资源
location ^~ /images/ {
    root /www/wwwroot/xuexitong/intro;
    expires 30d;
    access_log off;
    add_header Cache-Control "public";
}
```

**关键点说明：**
- `location = /` 是精确匹配，只匹配根路径 `/`，优先级高于 `location /` 反向代理
- `location ^~ /images/` 前缀匹配，优先级也高于 `location /`，确保图片走静态文件
- 其他路径（/admin、/user、/api/* 等）不受影响，仍走后端代理

### 第三步：重载 Nginx

宝塔面板中点击「重载配置」，或 SSH 执行：
```bash
nginx -t && nginx -s reload
```

### 第四步：验证

- 访问 `https://xs.openget.cn/` → 应显示产品介绍页
- 访问 `https://xs.openget.cn/admin` → 仍正常进入管理后台
- 访问 `https://xs.openget.cn/user.html` → 仍正常进入用户中心
- 访问 `https://xs.openget.cn/libs/vue/3.4.31/vue.global.prod.js` → 仍正常返回 JS 文件
