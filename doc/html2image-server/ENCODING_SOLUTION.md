# 项目启动脚本乱码问题解决方案

## 问题描述
您遇到的"乱码报错"问题通常是由于Windows命令行的编码设置不当导致的。

## 解决方案

### 方案1：使用已修复的start.bat文件
请使用我们提供的`start.bat`文件，它已经针对编码问题进行了优化：
- 使用代码页936（简体中文）而不是65001（UTF-8）
- 这样可以更好地兼容中文显示

### 方案2：手动设置命令行编码
如果您仍然遇到问题，请按以下步骤操作：

1. **右键点击命令提示符（CMD）**
2. **选择"属性"**
3. **切换到"字体"选项卡**
4. **选择"新宋体"或"等宽字体"**
5. **切换到"颜色"选项卡**
6. **将"屏幕文字"颜色设置为白色**
7. **点击"确定"保存设置**

### 方案3：使用PowerShell运行
1. **右键点击空白处**
2. **选择"在此处打开PowerShell窗口"**
3. **运行以下命令**：
   ```powershell
   .\start.bat
   ```

### 方案4：使用管理员权限运行
1. **右键点击start.bat**
2. **选择"以管理员身份运行"**

## 调试步骤

如果问题仍然存在，请按照以下步骤进行调试：

### 步骤1：检查Node.js安装
```cmd
node --version
```
如果显示版本号，说明Node.js已正确安装。

### 步骤2：检查端口占用
```cmd
netstat -an | find ":3000"
```
如果显示端口被占用，请关闭占用进程。

### 步骤3：查看日志文件
检查`logs\startup.log`文件，查看详细的错误信息。

## 文件列表

- `start.bat` - 已修复的启动脚本（推荐使用）
- `start_stable.bat` - 稳定版本
- `start_utf8.bat` - UTF-8编码版本
- `start_debug.bat` - 调试版本（显示详细信息）
- `stop.bat` - 停止脚本
- `server.js` - 测试服务器
- `package.json` - 项目配置
- `api-docs.html` - API文档页面

## 使用方法

1. 确保已安装Node.js
2. 在项目目录中运行 `npm install`
3. 运行 `start.bat` 启动项目
4. 查看控制台输出和日志文件
5. 在浏览器中访问 `http://localhost:3000`

## 常见错误及解决办法

### 错误1：编码错误
**现象**：中文显示为乱码
**解决**：使用`start.bat`文件，它已设置正确的编码

### 错误2：窗口一闪而过
**现象**：脚本执行后立即关闭
**解决**：在脚本末尾添加`pause`命令，或使用`start_debug.bat`查看详细信息

### 错误3：端口被占用
**现象**：提示端口3000已被占用
**解决**：使用任务管理器结束占用端口的进程，或修改配置文件中的端口号
