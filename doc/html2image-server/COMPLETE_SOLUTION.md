# 项目启动脚本乱码问题 - 完整解决方案

## 问题分析

从您上传的截图中可以看到，中文字符显示为乱码。这是Windows命令行编码设置导致的问题。

### 根本原因
1. Windows命令行默认编码可能与脚本编码不匹配
2. 批处理文件（.bat）的编码格式问题
3. 代码页（chcp）设置不当

## 解决方案

我已经为您创建了多个版本的启动脚本：

### 1. 推荐方案：使用英文版本（完全避免中文编码问题）

**文件**: `start_clean.bat`

**优点**: 
- 使用纯英文，完全避免中文编码问题
- 编码设置更稳定
- 适合所有Windows系统

**使用方法**:
1. 双击运行 `start_clean.bat`
2. 按照英文提示操作

### 2. 备选方案：使用中文版本（优化编码）

**文件**: `start_cn.bat`

**改进**:
- 使用代码页936（简体中文）
- 优化变量设置方式
- 更稳定的编码处理

**使用方法**:
1. 双击运行 `start_cn.bat`
2. 如果仍有问题，尝试方案1

### 3. 系统级解决方案

如果脚本仍然显示乱码，请按以下步骤操作：

#### 步骤1：修改命令行属性
1. 右键点击命令提示符（CMD）
2. 选择"属性"
3. 切换到"选项"选项卡
4. 确保"快速编辑模式"已勾选
5. 切换到"字体"选项卡
6. 选择"新宋体"或"Lucida Console"
7. 点击"确定"

#### 步骤2：设置代码页
在运行脚本前，先在命令行中运行：
```cmd
chcp 936
```

然后再运行启动脚本。

#### 步骤3：使用PowerShell
1. 在文件资源管理器中打开项目文件夹
2. 按住Shift键，同时右键点击空白处
3. 选择"在此处打开PowerShell窗口"
4. 运行：
```powershell
.\start_clean.bat
```

## 故障排除

### 问题1：仍然显示乱码
**解决方案**:
1. 使用 `start_clean.bat`（英文版）
2. 修改命令行字体为"新宋体"
3. 使用PowerShell运行

### 问题2：窗口快速关闭
**解决方案**:
脚本已添加 `pause` 命令，如果窗口仍然快速关闭，说明脚本可能在早期就遇到了错误。
请查看日志文件或使用调试版本 `start_debug.bat`。

### 问题3：Node.js未安装
**解决方案**:
1. 访问 https://nodejs.org
2. 下载并安装LTS版本
3. 重新运行启动脚本

### 问题4：端口被占用
**解决方案**:
1. 打开任务管理器
2. 结束占用端口3000的进程
3. 或修改脚本中的端口号
