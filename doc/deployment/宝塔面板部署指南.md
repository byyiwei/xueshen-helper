# 养龟档案 - 宝塔面板部署指南

## 📋 前置准备

| 项目 | 要求 |
|------|------|
| 服务器 | CentOS 7+ / Ubuntu 20.04+ |
| 宝塔面板 | 最新版（已安装） |
| 域名 | 已备案域名 + SSL证书 |

## 一、宝塔面板环境安装

### 1.1 安装 Node.js

登录宝塔面板 → **软件商店** → 搜索 "Node.js版本管理器" → 安装

安装后点击 **设置** → 安装 **Node.js 22.x LTS** 版本

### 1.2 安装 MySQL

宝塔面板 → **软件商店** → 搜索 "MySQL" → 安装 **MySQL 8.0**

安装完成后：
- 点击 MySQL → **性能调整**，根据服务器内存优化
- **root密码** 请设置为强密码并牢记

### 1.3 安装 PM2（通过 Node.js 管理器）

宝塔面板 → **软件商店** → 已安装的 Node.js版本管理器 → **设置** → **PM2管理**

或者在终端执行：
```bash
npm install -g pm2
```

### 1.4 安装 Nginx

宝塔面板默认已安装 Nginx。如未安装：
宝塔面板 → **软件商店** → 搜索 "Nginx" → 安装最新版

### 2.3 索引说明

数据库已预设以下核心索引，确保查询性能：

| 表 | 索引 | 用途 |
|----|------|------|
| users | `uk_openid` (唯一) | 登录查询 |
| users | `idx_status` | 封禁状态筛选 |
| admins | `uk_openid` + `uk_username` | 管理员权限校验/Web登录 |
| pets | `uk_openid_alias` (复合唯一) | 别名唯一性 |
| pets | `idx_openid` + `idx_is_public` | 用户宠物列表 |
| records | `idx_openid_pet` (复合) | 最高频查询：某用户某宠物 |
| records | `idx_pet_type` (复合) | 按类型筛选记录 |
| reminders | `uk_pet_type` (复合唯一) | 防止重复提醒 |
| security_logs | `idx_trace_id` | 微信回调匹配 |
| notifications | `idx_openid_unread` (复合) | 未读通知查询 |

## 五、Nginx 反向代理配置

### 5.1 添加站点

宝塔面板 → **网站** → **添加站点**：
- 域名：你的域名
- 根目录：`/www/wwwroot/你的域名`
- PHP版本：选择 **纯静态**

### 5.2 修改配置文件

宝塔面板 → **网站** → 点击域名 → **配置文件**，替换为以下内容：

```nginx
server {
    listen 80;
    server_name pets.openget.cn;
    
    # 日志（宝塔已配置）
    # access_log /www/wwwlogs/你的域名.log;
    
    client_max_body_size 20M;
    
    # API 服务反向代理
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # 静态文件：用户上传的图片
    location /uploads/ {
        alias /www/wwwroot/pets.openget.cn/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        add_header Access-Control-Allow-Origin *;
    }
    
    # 健康检查
    location /health {
        access_log off;
        return 200 "ok";
    }
}
```

### 5.3 配置 SSL（推荐）

宝塔面板 → **网站** → 点击域名 → **SSL** → 选择 **Let's Encrypt** → **申请证书**

然后修改 Nginx 配置，在 80 端口 server 块之前添加 HTTPS 配置：

```nginx
server {
    listen 443 ssl http2;
    server_name pets.openget.cn;
    
    # SSL 证书（宝塔自动填写）
    ssl_certificate /www/server/panel/vhost/cert/pets.openget.cn/fullchain.pem;
    ssl_certificate_key /www/server/panel/vhost/cert/你的域名/privkey.pem;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    client_max_body_size 20M;
    
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
        
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    location /uploads/ {
        alias /www/wwwroot/pets.openget.cn/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        add_header Access-Control-Allow-Origin *;
    }
}

# HTTP 自动跳转 HTTPS
server {
    listen 80;
    server_name pets.openget.cn;
    return 301 https://$host$request_uri;
}
```

完成后点击 **保存**，然后 **重载配置**。

## 六、微信小程序配置

### 6.1 配置服务器域名

登录 [微信公众平台](https://mp.weixin.qq.com) → **开发管理** → **开发设置** → **服务器域名**：

| 类型 | 域名 |
|------|------|
| request合法域名 | `https://你的域名.com` |
| uploadFile合法域名 | `https://你的域名.com` |
| downloadFile合法域名 | `https://你的域名.com` |

### 6.2 配置消息推送（审核回调）

在微信公众平台 → **开发管理** → **开发设置** → **消息推送**：
- URL：`https://你的域名.com/api/callback/security`
- Token：自定义随机字符串
- 消息加密方式：明文模式

### 6.3 修改小程序 API 地址

在小程序 `app.js` 或首次加载时设置：

```javascript
// 在 globalData.systemConfig 中设置：
globalData: {
  systemConfig: {
    apiUrl: 'https://你的域名.com',
    // 或通过 GET /api/admin/config 从服务器动态加载
  }
}
```

```bash

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

## 九、故障排查

| 问题 | 检查项 |
|------|--------|
| 502 Bad Gateway | `pm2 status` 检查服务是否运行 |
| API 请求超时 | 检查数据库连接、防火墙是否开放 3000 |
| 图片上传失败 | 检查 `/uploads` 目录权限和磁盘空间 |
| 微信登录失败 | 检查 AppID/AppSecret 是否正确 |
| 审核回调无响应 | 检查微信公众平台消息推送配置 |
| 数据库连接失败 | 检查 MySQL 是否运行、用户名密码是否正确 |
